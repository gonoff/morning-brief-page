CT Pros ERP

CT Pros · Laravel 11 + Inertia + Vue 3 + QuickBooks
9:00 – 10:00 M · 60 min t29 · HIGH · CRITICAL before prod
Sandbox-validate all QuickBooks field mappings
What
Run every QB sync() method against the Intuit sandbox and verify that every field name, data type, and relationship matches the real QBO API before this ever touches a live client account.
Understand
Intuit's sandbox is a fully isolated QB company — changes there never reach real data. The 6 entity syncs (customers, invoices, payments, vendors, bills, expense accounts) each have their own QBO endpoint and their own field quirks (e.g. CustomerRef vs CustomerRefFull, TxnDate vs DocNumber). A mapping that looks right in code can silently write garbage in prod.
Approach
  1. Boot your dev env pointed at Intuit sandbox credentials (check .env.testing — QB_CLIENT_ID, QB_SANDBOX=true).
  2. Trigger each of the 6 DataService sync() methods one by one. Read the raw QBO response and diff field names against what your code writes.
  3. Fix any mismatch (null fields, wrong type, missing required field). Keep a running list.
  4. Add a PHPUnit test per entity that mocks DataService and asserts the payload shape — locks the contract so regressions are caught automatically.
Resources
  • Intuit Developer Sandbox: developer.intuit.com → your sandbox company
  • QBO API Reference: developer.intuit.com/app/developer/qbo/docs/api/accounting
  • app/Services/QuickBooks/DataService.php — your sync methods
  • PHPUnit + Mockery::mock() or Laravel's $this->mock()
  • Postman collection if you have one — fire raw API calls to see exact QB responses

11:30 – 12:30 M · 60 min t26 · HIGH · Harborview demo failure
Label Lane v2 — productize M3 tiles as opt-in AI label mode
What
Turn the M3 tiles label-reading prototype that crashed the Harborview demo into a stable, opt-in API endpoint: /sheets/{id}/read-labels?mode=ai, backed by MINIMAX_AI, with callout filtering to kill the false positives.
Understand
The demo failed because the AI lane was active by default and wasn't production-hardened. The two fixes: (1) make it strictly opt-in so standard requests are never affected, (2) filter callout boxes and dimension leaders that were firing as "labels" — they were the main source of noise. M3 tiles are the PDF rendered at zoom level 3 and sliced into grid cells.
Approach
  1. Add mode: str = "standard" query param to the endpoint. Early-return the standard result if mode != "ai".
  2. Add MINIMAX_AI_KEY to the VPS .env and read it in settings.py.
  3. In the AI label lane, add a callout filter: reject any detected text region whose bounding box aspect ratio is > 4:1 (callout leaders are thin horizontal strips) and any region touching a zone boundary (those are dimensions, not room labels).
  4. Test with the Harborview sheet as your regression fixture — confirm only room labels survive.
Resources
  • FastAPI query param: async def read_labels(sheet_id: str, mode: str = "standard")
  • MINIMAX AI: check your existing integration for the API call pattern
  • Live service: aztakeoff.azteamai.com
  • VPS: SSH in, add env var, systemctl restart takeoff-engine
  • Harborview test sheet — your regression fixture

Assemblies-per-room layer — classification → wall quantities → CSI formulas → new Excel tab
What
Add a full assemblies layer: take each detected room's classification, compute wall quantities (perimeter × height), apply CSI MasterFormat division formulas, and write everything to a new "Assemblies" tab in the Excel output.
Understand
CSI MasterFormat organizes construction materials into numbered divisions (Division 09 = Finishes: drywall, paint, flooring — your most common). Each room type (office, corridor, bathroom) has a standard assembly — a set of finish types that are presumed. Your engine already has room sqft and perimeter from the detection layer; this task layers material quantities on top. Default wall height: 9 ft commercial, 8 ft residential.
Approach
  1. Define an ASSEMBLY_RULES dict: room_type → list of {csi_code, item, unit, qty_formula}. Start with 5–6 common room types; you can expand later.
  2. For each room: compute wall_sf = perimeter_ft * wall_height_ft, ceiling_sf = room_sqft. Apply rules to get per-item quantities.
  3. Aggregate across all rooms — same CSI item from multiple rooms sums together into one row.
  4. Add an "Assemblies" worksheet to the openpyxl workbook: columns = CSI Code | Item | Unit | Qty | Notes. Freeze row 1. Add a subtotal row per division group.
Resources
  • CSI MasterFormat 2016 — focus on Div 03 (Concrete), Div 06 (Wood), Div 09 (Finishes)
  • openpyxl: wb.create_sheet("Assemblies"), ws.freeze_panes = "A2"
  • Your existing takeoff/export.py (or wherever Excel generation lives) — add the new sheet there
  • Room classification enum from your detection layer — keys in ASSEMBLY_RULES must match exactly
  • Wall height config: add WALL_HEIGHT_FT: float = 9.0 to settings.py
